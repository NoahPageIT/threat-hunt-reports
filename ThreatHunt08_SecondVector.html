// CONTROL GAPS IDENTIFIED
GAP 01 // CONDITIONAL ACCESS NOT ENFORCED ON LEGACY AUTH

All sign-ins from 103.69.224.136 returned ConditionalAccessStatus = "notApplied" with AuthenticationProtocol = "none". Legacy authentication protocols fall completely outside CA policy scope unless explicitly blocked.

Block legacy authentication via Conditional Access policy targeting all users. Use the "Block legacy authentication" template in Entra ID.
GAP 02 // POWER AUTOMATE DLP GOVERNANCE GAP

Attacker created a fully functional exfiltration flow via Microsoft Power Automate using delegated credentials. No DLP policy blocked the flow from sending to external mail destinations.

Implement Power Platform DLP policies. Restrict Office 365 Mail connector from sending to external business group connectors.
GAP 03 // SESSION PERSISTENCE AFTER PASSWORD RESET

Active refresh tokens survive a password reset. An attacker with a valid token retains full access even after the victim's password is changed — a critical containment gap.

Always revoke active sessions FIRST (Revoke-AzureADUserAllRefreshToken), then reset password. Session revocation is a separate, mandatory step.

// LESSONS LEARNED
›

Run the broad search pivot first. A targeted union query across manually selected tables missed 2 of 7 sources. The search operator across all workspace tables is essential before building targeted queries.

›

ConditionalAccessStatus = "notApplied" is the signal. It means a sign-in occurred completely outside your CA policy scope — legacy auth is a persistent blind spot that bypasses MFA enforcement entirely.

›

Power Platform is an exfiltration vector. Power Automate with delegated user credentials bypasses many email DLP controls. Governance policies on connectors are mandatory, not optional.

›

Password reset ≠ containment. Active refresh tokens survive password resets. Revoke sessions first, then reset — containment is incomplete without both steps, in that order.
